## English

Software development is more than coding. Architecture is more than a polished UI.

While LinkedIn is full of "weekend MVPs" and quickly assembled AI frontends, we have been building on a different layer: Today I am releasing Nova-shell v0.8.13.

Nova-shell is not just another framework. It is a declarative runtime, shell, and distributed AI OS platform with its own `.ns` language.

Why this release matters to me:

✅ Real engineering quality instead of demo theater  
217 automated unit and integration tests cover the parser, runtime, CLI, mesh, and release paths across Python, C++, GPU, and WASM. A clean run in around 91 seconds is not marketing. It is a concrete quality signal.

✅ Security as a foundational layer  
Nova-shell includes its own internal CA, trust policies, and mTLS onboarding for mesh workers. Security is not an afterthought here. It is part of the architecture.

✅ Event-driven orchestration instead of fragile agent loops  
Flows, events, state, queueing, replay, snapshots, and consensus mechanisms form a controllable runtime. The goal is not a flashy demo moment, but reproducible behavior in distributed systems.

✅ Documentation as part of the platform  
With `wiki build`, Nova-shell generates its HTML wiki directly from the project itself. Documentation is not detached overhead. It is part of the system.

To me, the important line today is no longer between human and machine. It is between superficial prototypes and real system architecture.

Nova-shell is my attempt to bring language, runtime, AI, knowledge, and distributed orchestration into one coherent platform.
